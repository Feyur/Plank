# Решения

Коротко: что решили, почему, какие были альтернативы. Одно решение — один абзац.
Новые — сверху.

**2026-07-16 · Упоминания — точный `@ник` в комментарии карточки.** У каждого
пользователя есть уникальный ник из букв, цифр и `_`; его можно задать при
создании аккаунта и поменять в профиле. При отправке комментария сервер ищет
точные `@ники` только среди тех, кто сейчас видит эту доску, не уведомляет
автора и сохраняет одну запись `notifications` на получателя и комментарий.
Колокольчик получает обновление через существующий WebSocket-сигнал и
перезапрос, показывает число непрочитанных и помечает их прочитанными при
открытии. От push/email-уведомлений пока отказались: для маленькой команды
достаточно in-app сценария без внешнего сервиса.

**2026-07-16 · Доступ к доскам назначается аккаунту явно.** Таблица
`board_members` связывает пользователя с видимыми досками; проверка выполняется
на API для досок и всех дочерних ресурсов. Аккаунты из `ACCESS_ADMIN_EMAILS`
(по умолчанию `vip@atank.ru` и `web@atank.ru`) видят все доски, могут создавать
пользователей с email, паролем, ником и выбранными досками, а также менять набор
досок уже добавленного участника. Существующие связи заполняются при миграции,
поэтому текущие пользователи не теряют доступ; новая доска автоматически доступна
своему создателю. От полноценных ролей и workspaces пока отказались: для текущей
задачи достаточно двух уровней — администратор доступа и участник.

**2026-07-11 · Тёмная тема — палитра выведена вручную.** Готовых тёмных значений
не было, а тема нужна была сейчас. Сделали в духе светлой палитры: весь блок
`[data-theme='dark']` в `apps/web/src/tokens.css` — производный. Механика: data-theme на `<html>`, localStorage, дефолт —
prefers-color-scheme, тумблер в меню пользователя. Все цвета компонентов
предварительно сведены в токены (включая палитру меток `--chip-*`), так что
замена на официальные значения — правка одного файла.

**2026-07-11 · HTTPS на нестандартном порту 9443.** На vps3 порты 443 и 8443
заняты Shadowsocks (маскировка VPN под HTTPS — двигать нельзя, сломается
маскировка и клиенты). Caddy: сертификат Let's Encrypt по HTTP-01 на :80,
HTTPS на :9443, http→https-редирект вручную (авто-редирект вёл бы на чужой :443).
Альтернативы: подвинуть SS (риск), отдельный сервер (вариант на будущее).

**2026-07-11 · Прод: один контейнер app (фронт+API+WS) + Postgres + Caddy.**
Node отдаёт собранную статику через @fastify/static — нет отдельного
nginx/фронт-контейнера, меньше RAM (всё вместе ~80 МБ). Образ собирается
локально под linux/amd64 и переносится `docker save | ssh docker load` —
на VPS с 1 ГБ не собираем. Миграции и сидинг аккаунтов — в CMD на старте.

**2026-07-11 · Регистрация закрыта на проде.** На production регистрация
закрыта по умолчанию и может быть открыта только явным
`REGISTRATION_ENABLED=true`; административные email через неё создать нельзя.
Основной путь — форма управления доступами, `SEED_USERS` остаётся для первого
запуска.

**2026-07 · Одна команда на инстанс, без workspaces.** Пространство команды
остаётся единым, но с 2026-07-16 видимость досок ограничивается через
`board_members`. `owner_id` по-прежнему означает только «кто создал».
Workspaces и сложные роли отложены по YAGNI до появления второй команды.

**2026-07 · Realtime = «сигнал + перезапрос», не патчи состояния.** Любая
успешная мутация доски шлёт всем `board-changed` (onResponse-хук), клиент
тихо перезапрашивает доску (debounce 250 мс). Проще и надёжнее CRDT/патчей;
для команды в 2–10 человек трафик незаметен. Пересмотреть при росте.

**2026-07 · Порядок через дробный position.** Доски/колонки/карточки/чек-лист
хранят `position double precision`; вставка = середина между соседями, без
перенумерации. Известный предел — точность float после сотен вставок в одну
точку (в бэклоге компактизация).

**2026-07 · Чат доски — плоский поток, «прочитано» на клиенте.** Таблица
`board_messages` без тредов и реакций (быстрые вопросы, не мессенджер).
Отметка «прочитано» — в localStorage на каждую доску: не тащим в БД состояние,
которое стоит терять безболезненно. Realtime — тот же «сигнал → перезапрос».

**2026-07 · Папки досок — отдельная сущность `folders` (было: строка).**
Начали со строки `boards.folder`, но UX потребовал пустых папок, цвета и
перетаскивания — это невозможно на строковом поле. Завели таблицу `folders`
(имя, цвет, порядок) и `boards.folder_id` (on delete set null). Доска кладётся
в папку drag-and-drop'ом (dnd-kit: папки — droppable-зоны). window.prompt для
ввода имени убрали — он не работает в Tauri-обёртке.

**2026-07 · Дейли — командный, одна запись на человека в день.** Таблица
`daily_entries` c `unique(user_id, entry_date)`; три раздела (готово/в работе/
планирую) как текст, пункт на строку. Все видят всех (командный стендаб),
править можно только свою запись. Дизайн-прототип был просмотровым — добавили
редактирование своей записи.

**2026-07 · Время срока карточки — отдельное поле `due_time`, не timestamptz.**
Дату держим строкой-датой (без TZ-сдвигов), поэтому необязательное время —
отдельным `text 'HH:MM'`. В дизайне времени у срока не было — это расширение.

**2026-07 · Метки — строки в БД, а не enum.** `labels` на доску (имя + ключ
цвета из фиксированной палитры токенов), карточка ссылается по id. Палитра
цветов фиксирована в коде/токенах — свобода выбора цвета не нужна, консистентность важнее.

**2026-07 · Сессия — JWT в httpOnly-cookie на 30 дней.** Без таблицы сессий:
меньше кода и запросов. Минус — нельзя отозвать до истечения (в бэклоге
token_version). SameSite=Lax, Secure на HTTPS.

**2026-06 · Стек: React+Vite / Fastify / Postgres / свой WebSocket.** Лёгкий
самодостаточный стек под VPS ~1 ГБ RAM. Отказались от self-hosted Supabase
(7–8 контейнеров, не влезает) и облачных BaaS (внешняя зависимость).
